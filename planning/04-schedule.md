# Energy Budget - 6주 일정

## 1. 문서 목적

이 문서는 Energy Budget 프로젝트의 10주차부터 15주차까지의 개발 일정을 정리한다. 일정은 Week 10 수업 기준에 맞춰 기획, 설계, 핵심 기능, 테스트, 배포/문서, 최종 발표 흐름으로 작성한다.

## 2. 6주 일정 요약

| 주차 | 목표 | 주요 산출물 | 검증 기준 |
|---|---|---|---|
| 10주차 | 기획과 일정 수립 | `.planning/00-vision.md`, `.planning/01-requirements.md`, `.planning/02-wbs.md`, `.planning/03-risks.md`, `.planning/04-schedule.md` | 프로젝트 목표, 사용자 시나리오, Must 기능을 설명할 수 있음 |
| 11주차 | 설계와 환경 구축 | `docs/architecture.md`, `docs/setup.md`, ADR 문서, Expo Hello World | `npm install` 후 `npm run start` 실행 가능 |
| 12주차 | 핵심 기능 1 구현 | 컨디션 입력 화면, 할 일 등록 화면, 에너지 계산 규칙 | 사용자가 컨디션과 할 일을 입력하고 에너지 예산을 확인할 수 있음 |
| 13주차 | 핵심 기능 2 구현과 테스트 | 오늘 할 일 추천 화면, 일일 기록 저장, 완료 여부 체크 | 에너지 예산 안에서 추천 결과가 표시되고 기록 저장 가능 |
| 14주차 | 리포트, 마감, 문서 정리 | 주간 리포트 화면, README, 발표용 화면 캡처, 테스트 목록 | 핵심 사용자 시나리오 3개를 앱에서 시연 가능 |
| 15주차 | 최종 발표 | 발표 자료, 회고, 최종 데모 | 문제, 해결 방법, 기술 선택, 결과물을 발표할 수 있음 |

## 3. 주차별 상세 계획

### 10주차. 기획과 일정 수립

목표:
- Energy Budget의 문제 정의와 핵심 가치를 정리한다.
- 사용자 시나리오 3개를 작성한다.
- 기능을 MoSCoW 방식으로 분류한다.
- WBS와 위험 대응, 6주 일정을 작성한다.

완료 기준:
- 프로젝트 한 줄 소개를 말할 수 있다.
- Must 기능 5개를 설명할 수 있다.
- 가장 큰 위험과 대응 방안을 설명할 수 있다.

### 11주차. 설계와 환경 구축

목표:
- 모바일 플랫폼 선택 이유를 ADR로 정리한다.
- Presentation, Application, Domain, Data 레이어 구조를 만든다.
- Expo 기반 Hello World 화면을 실행 가능한 상태로 만든다.
- `docs/setup.md`, `docs/architecture.md`를 작성한다.

완료 기준:
- ADR-0001 ~ ADR-0003을 설명할 수 있다.
- 새 화면, 계산 규칙, 저장 코드를 어느 폴더에 둘지 말할 수 있다.
- `npm run start`로 앱을 실행할 수 있다.

### 12주차. 핵심 기능 1 구현

목표:
- 오늘의 컨디션 입력 화면을 구현한다.
- 할 일 등록 화면을 구현한다.
- 에너지 예산 계산 규칙을 구현한다.

완료 기준:
- 사용자가 피로도, 기분, 남은 시간을 입력할 수 있다.
- 사용자가 할 일, 예상 시간, 에너지 소모량을 등록할 수 있다.
- 입력값을 바탕으로 오늘의 에너지 예산이 계산된다.

### 13주차. 핵심 기능 2 구현과 테스트

목표:
- 오늘 할 일 추천 기능을 구현한다.
- 미루기 후보 목록을 보여준다.
- 일일 계획과 완료 여부를 저장한다.

완료 기준:
- 에너지 예산 안에서 실행 가능한 할 일이 추천된다.
- 에너지 예산을 넘는 일은 미루기 후보로 구분된다.
- 사용자는 완료 여부를 체크하고 일일 기록을 저장할 수 있다.

### 14주차. 리포트와 문서 정리

목표:
- 주간 에너지 리포트 화면을 만든다.
- 최근 7일의 컨디션, 완료율, 미룬 일을 요약한다.
- README와 발표 자료 초안을 정리한다.

완료 기준:
- 사용자는 자신의 일주일 에너지 사용 패턴을 볼 수 있다.
- 핵심 사용자 시나리오 3개를 앱에서 시연할 수 있다.
- 실행 방법과 주요 기능이 문서화되어 있다.

### 15주차. 최종 발표

목표:
- 최종 발표 자료를 완성한다.
- 프로젝트 문제, 해결 방법, 기술 선택, 구현 결과를 정리한다.
- 회고와 개선 방향을 작성한다.

완료 기준:
- 앱 데모를 안정적으로 보여줄 수 있다.
- 왜 Energy Budget을 만들었는지 설명할 수 있다.
- 왜 React Native + Expo를 선택했는지 설명할 수 있다.

## 4. 일정 관리 원칙

- Must 기능은 13주차까지 끝내는 것을 목표로 한다.
- Should 기능은 14주차에 가능한 범위에서 구현한다.
- Could 기능은 발표 안정성이 확보된 뒤에만 추가한다.
- Won't 기능은 이번 프로젝트 범위에서 제외한다.
