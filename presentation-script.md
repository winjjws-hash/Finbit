# Energy Budget 2분 발표 대사

## 발표 전 준비

PC에서 발표 URL을 먼저 열어둔다. 스마트폰 타이머를 2분으로 맞춘다. 화면이 준비되면 타이머를 시작하고 발표한다.

발표 URL

```text
https://winjjws-hash.github.io/Finbit/presentation.html
```

## 0:00 - 0:20 기획 비전

안녕하세요. 제 프로젝트는 Energy Budget입니다.

이 앱은 시간표가 아니라 오늘의 에너지를 기준으로 하루 계획을 세우는 모바일 앱입니다.

제가 잡은 비전은 무리하지 않아도 꾸준히 성장할 수 있는 지속 가능한 일상을 만드는 것입니다.

## 0:20 - 0:45 문제 정의와 가치

기존 투두앱은 해야 할 일을 많이 적고 완료했는지만 확인합니다.

하지만 실제로는 시간이 있어도 피곤하거나 기분이 좋지 않으면 계획을 지키기 어렵습니다.

그래서 저는 문제를 시간 부족이 아니라 에너지 관리 부족으로 봤습니다.

Energy Budget은 피로도, 기분, 남은 시간을 입력하면 오늘 감당 가능한 일을 추천해서 무리한 계획을 줄여줍니다.

## 0:45 - 1:10 WBS 일괄 확인

WBS는 GitHub Pages에서 한 번에 확인할 수 있게 정리했습니다.

10주차에는 비전, 요구사항, 기능 분류, WBS를 만들었습니다.

11주차에는 ADR, 아키텍처, Expo 환경, GitHub Pages를 완성했습니다.

12주차부터는 컨디션 입력, 할 일 등록, 에너지 계산 기능을 실제 화면과 연결할 계획입니다.

## 1:10 - 1:35 설계 아키텍처

설계는 Mermaid 구조로 표현했습니다.

사용자가 보는 화면은 presentation, 화면에 필요한 흐름은 application, 에너지 계산 규칙은 domain, 기록 저장은 data로 나눴습니다.

이렇게 나눈 이유는 화면 코드와 계산 규칙이 섞이지 않게 해서 나중에 수정하거나 설명하기 쉽게 만들기 위해서입니다.

## 1:35 - 2:00 현재 결과와 마무리

현재는 비전, 사용자 시나리오, MoSCoW, WBS, ADR, 아키텍처 문서, 발표 페이지까지 완료했습니다.

다음 단계에서는 컨디션 입력과 할 일 등록, 에너지 예산 계산을 실제 기능으로 연결할 예정입니다.

정리하면 Energy Budget은 더 많이 하게 만드는 앱이 아니라 오늘의 나에게 맞는 계획을 세우게 돕는 앱입니다.

감사합니다.
