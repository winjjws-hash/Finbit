# Energy Budget - BONUS 초안

## 1. 목적

이 문서는 Energy Budget 프로젝트에서 가산점을 노릴 수 있는 요소를 정리하기 위한 초안이다. 실제 구현 상황에 따라 체크 상태를 업데이트한다.

## 2. 가산점 후보

| 항목 | 설명 | 현재 상태 | 우선순위 |
|---|---|---|---|
| ADR 3개 이상 작성 | 플랫폼, 상태관리, 백엔드/저장 방식, 프로젝트 주제 선택 이유를 문서화 | 진행 중 | 높음 |
| 아키텍처 문서 작성 | 레이어드 구조와 파일 위치를 쉽게 설명하는 문서 작성 | 진행 중 | 높음 |
| 실행 가이드 작성 | 새 사람이 `npm install`, `npm run start`로 실행할 수 있게 안내 | 진행 중 | 높음 |
| 사용자 시나리오 기반 테스트 | 아침 계획, 중간 조정, 주간 회고 시나리오로 앱 테스트 | 예정 | 높음 |
| 주간 리포트 기능 | 에너지 기록과 완료율을 요약해서 보여주는 화면 | 예정 | 중간 |
| 발표용 60초 설명 준비 | 주제, 기술 선택, 구조를 짧게 설명하는 문장 준비 | 진행 중 | 중간 |
| 데이터 내보내기 | 기록을 CSV 또는 Markdown으로 내보내는 기능 | 선택 | 낮음 |

## 3. 우선으로 노릴 가산점

이번 프로젝트에서 우선으로 노릴 가산점은 다음 세 가지이다.

1. **ADR 문서화**
   - 프로젝트 주제, 플랫폼, 상태관리, 저장 방식 선택 이유를 남긴다.

2. **실행 가능한 MVP**
   - `git clone` 후 한 줄 명령으로 실행 가능한 상태를 목표로 한다.

3. **사용자 시나리오 기반 발표**
   - 단순 기능 나열이 아니라 실제 사용 흐름으로 시연한다.

## 4. 발표에서 강조할 점

Energy Budget은 단순한 투두앱이 아니라, 사용자의 하루 컨디션과 에너지를 기준으로 현실적인 계획을 추천하는 앱이다. 따라서 가산점 요소도 기능을 많이 넣는 것보다, 문제 정의와 사용 흐름을 명확하게 보여주는 방향으로 준비한다.
