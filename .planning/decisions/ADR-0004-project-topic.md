# ADR-0004. 프로젝트 주제 선택

## 상태

Accepted

## 배경

이번 프로젝트는 6주 안에 혼자 기획, 설계, 구현, 발표까지 진행해야 한다. 또한 바이브코딩을 활용하되, 결과물을 본인이 이해하고 설명할 수 있어야 한다.

처음에는 데이터 분석, 개발자 회고, 실생활 도구 등 여러 주제를 검토했다. 그중 Energy Budget은 실제 생활에서 자주 사용할 수 있고, 발표와 포트폴리오에도 활용 가능한 주제라고 판단했다.

## 문제

많은 일정 관리 앱은 사용자가 해야 할 일을 많이 등록하고 완료 여부를 체크하는 데 집중한다. 하지만 실제로는 시간이 있어도 피곤하거나, 기분이 좋지 않거나, 하루 에너지가 부족해서 계획을 지키지 못하는 경우가 많다.

이 문제는 단순히 의지가 부족해서 생기는 문제가 아니라, 하루 계획을 세울 때 자신의 컨디션과 에너지 상태를 충분히 고려하지 않기 때문에 발생한다.

## 결정

프로젝트 주제로 **Energy Budget: 하루 에너지 예산 관리 앱**을 선택한다.

Energy Budget은 사람들의 하루 계획을 시간 중심에서 에너지 중심으로 바꾸어, 무리하지 않아도 꾸준히 성장할 수 있는 지속 가능한 일상을 만드는 모바일 앱이다.

## 결정 이유

1. 실생활에서 자주 사용할 수 있다.
   - 사용자는 매일 아침이나 하루 중간에 자신의 컨디션을 확인하고 계획을 조정할 수 있다.

2. 기존 투두앱과 차별점이 있다.
   - 단순히 할 일 목록을 보여주는 것이 아니라, 오늘 감당 가능한 에너지 기준으로 할 일을 추천한다.

3. 6주 안에 MVP 구현이 가능하다.
   - 컨디션 입력, 할 일 등록, 에너지 계산, 추천 결과 표시, 기록 저장까지 작은 단위로 나눠 만들 수 있다.

4. 발표하기 쉽다.
   - "시간은 있는데 왜 계획을 못 지킬까?"라는 문제에서 출발하기 때문에 청중이 쉽게 공감할 수 있다.

5. 포트폴리오에 작성하기 좋다.
   - 문제 정의, 사용자 시나리오, 추천 로직, 모바일 UI, 아키텍처 설계, ADR 기록까지 보여줄 수 있다.

## 대안

| 대안 주제 | 장점 | 선택하지 않은 이유 |
|---|---|---|
| 개발자 감정 로그북 | IT 전공자에게 잘 맞고 구현이 쉬움 | 개발 기록/회고 중심이라 실생활 사용 범위가 좁음 |
| 소음 지도 | 독창적이고 장소 추천 기능이 흥미로움 | 사용 빈도가 특정 장소 방문 상황에 제한됨 |
| 대기시간 기록 앱 | 데이터 기록과 예측 요소가 있음 | 사용자가 꾸준히 대기시간을 입력해야 해서 지속 사용 부담이 큼 |
| 소비 후회 기록장 | 가계부와 차별화 가능 | 구매 후 기록 습관이 필요하고 프로젝트 방향이 소비 관리로 좁아짐 |
| 냉장고 유통기한 앱 | 실용성이 높음 | 요리나 장보기 습관이 없는 사용자에게 사용성이 낮음 |

## 결과

- 프로젝트의 핵심 가치는 "더 많이 하기"가 아니라 "무리하지 않고 꾸준히 하기"로 정한다.
- MVP의 핵심 기능은 오늘의 컨디션 입력, 할 일 등록, 에너지 예산 계산, 오늘 할 일 추천, 일일 기록 저장으로 정한다.
- 플랫폼은 모바일 사용성을 고려해 React Native + Expo를 사용한다.
- 초기 범위에서는 외부 캘린더 연동, 웨어러블 기기 연동, AI 상담 챗봇은 제외한다.

## 발표용 60초 설명

우리 프로젝트는 Energy Budget이라는 하루 에너지 예산 관리 앱입니다. 기존 일정 관리 앱은 할 일을 얼마나 많이 처리했는지에 집중하지만, 실제 생활에서는 시간이 있어도 피곤해서 계획을 지키지 못하는 경우가 많습니다. 그래서 이 앱은 사용자의 피로도, 기분, 남은 시간을 바탕으로 오늘 감당 가능한 에너지 예산을 계산하고, 그 안에서 할 일을 추천합니다. 대안으로 개발자 감정 로그북이나 소음 지도 같은 주제도 고민했지만, Energy Budget이 실생활에서 더 자주 쓸 수 있고 6주 안에 MVP로 만들기 적합하다고 판단했습니다.
