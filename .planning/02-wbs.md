# Energy Budget - WBS

## 1. 문서 목적

이 문서는 Energy Budget 프로젝트의 작업을 산출물 중심으로 분해한 WBS이다. 각 항목은 1~3일 안에 작업 가능한 크기를 기준으로 작성한다.

## 2. WBS 작성 기준

- 작업은 동사형 활동이 아니라 명사형 산출물로 표현한다.
- 최대 3단계 깊이까지만 분해한다.
- MVP 구현에 필요한 Must 기능을 우선으로 배치한다.
- Should 기능은 핵심 기능 완성 후 추가 가능한 범위로 배치한다.
- Won't 기능은 이번 WBS에 포함하지 않는다.

## 3. WBS 요약

| WBS | 산출물 | 설명 | 예상 기간 | 우선순위 |
|---|---|---|---:|---|
| 1 | 기획 산출물 | 비전, 요구사항, WBS, 일정 문서 | 2일 | Must |
| 2 | 모바일 화면 설계 산출물 | 주요 화면 구조와 사용자 흐름 | 2일 | Must |
| 3 | 데이터 구조 산출물 | 컨디션, 할 일, 기록 데이터 구조와 로컬 저장 방식 | 2일 | Must |
| 4 | 핵심 입력 기능 | 컨디션 입력과 할 일 등록 기능 | 3일 | Must |
| 5 | 에너지 계산 기능 | 에너지 예산 계산과 과부하 판단 | 3일 | Must |
| 6 | 추천 기능 | 오늘 할 일 추천과 미루기 후보 표시 | 3일 | Must |

| 8 | 리포트 기능 | 주간 에너지 리포트와 기본 통계 | 3일 | Should |
