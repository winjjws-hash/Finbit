# Energy Budget 보안 체크리스트

## 현재 보안 범위

Energy Budget MVP는 외부 API, 로그인, 결제, 위치 정보, 카메라 권한을 사용하지 않습니다. 그래서 현재 보안의 핵심은 환경 변수 관리, 로컬 저장 계획, 입력 검증, 로그 관리입니다.

## 체크리스트

| 항목 | 상태 | 프로젝트 적용 |
|---|---|---|
| API 키 하드코딩 없음 | 통과 | 현재 외부 API를 사용하지 않음 |
| `.env` 커밋 방지 | 통과 | `.gitignore`에 `.env`, `.env.dev`, `.env.prod` 포함 |
| 인증서 커밋 방지 | 통과 | keystore, p8, p12, pem 확장자 제외 |
| 사용자 입력 검증 | 부분 통과 | 할 일 제목 빈 값 방지, 시간 범위 제한 |
| HTTPS 강제 | 해당 없음 | 현재 외부 통신 없음 |
| 로컬 민감정보 암호화 | 다음 단계 | SQLite 저장 시 민감정보 저장하지 않는 방향 |
| 로그에 토큰 출력 없음 | 통과 | 토큰, 비밀번호 사용 없음 |
| 권한 사유 명시 | 해당 없음 | 카메라, 위치, 알림 권한 사용 없음 |

## 입력 검증

현재 앱은 다음 입력을 제한합니다.

- 할 일 제목이 비어 있으면 추가하지 않습니다.
- 남은 시간은 0부터 16시간 사이로 계산합니다.
- 예상 시간은 5분부터 240분 사이로 제한합니다.
- 에너지 소모량은 선택 버튼으로 입력합니다.

## 로컬 저장 계획

14주차 기준 DB는 아직 실제 연결하지 않았습니다. 다음 단계에서 SQLite를 붙일 때 아래 원칙을 적용합니다.

- 이름, 토큰, 위치 같은 민감정보 저장 금지
- 하루 계획과 완료 여부만 저장
- 삭제 또는 초기화 기능 제공
- 백업 파일에 개인 데이터 포함 여부 확인

## Q&A 답변

보안 질문을 받으면 이렇게 답합니다.

> 현재 MVP는 외부 API와 로그인 기능이 없어서 API 키나 개인정보 유출 위험은 낮습니다. 대신 환경 변수와 인증서가 Git에 올라가지 않도록 `.gitignore`를 정리했고, 사용자 입력은 빈 값과 숫자 범위를 검증했습니다. 이후 SQLite 저장을 붙일 때도 민감정보는 저장하지 않는 방향으로 설계할 예정입니다.
